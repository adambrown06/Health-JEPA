================================================================================
                    Health-JEPA — Project status (concise)
        Action-conditional JEPA for clinical time series + platform roadmap
================================================================================


1. Overall summary
--------------------------------------------------------------------------------

  Health-JEPA is a causal healthcare analytics platform. The core is a Joint-
  Embedding Predictive Architecture (JEPA) on irregular clinical time series:
  it learns a latent representation of a patient’s pre-intervention trajectory
  and predicts a post-intervention representation under a discrete
  intervention z — in embedding space (MSE on latent vectors), with no
  reconstruction loss on raw labs.

  Downstream research artifacts include vector search (Qdrant “twins”) and full
  evaluation suites under ml/. An older full-stack prototype (Next.js, FastAPI,
  Celery, LLM/RAG) is archived on git branch archive/legacy-monorepo only.


2. All of Us (AoU) data collection (work completed)
--------------------------------------------------------------------------------

  A Researcher Workbench pipeline was implemented and run on the AoU OMOP CDR.
  The index event is the first drug exposure (OMOP drug_exposure, rolled up to
  ingredient level via concept_ancestor). A fixed cohort cap per arm was used
  to keep jobs tractable. Pre/post windows of ±365 days were extracted; signals
  were binned weekly to reduce size and memory.

  EHR: a curated set of labs and vitals (LOINC-aligned measurement concepts) —
  cardiometabolic and renal focus (e.g. HbA1c, lipids, creatinine, eGFR, blood
  pressure, BMI, SpO2).

  Wearables: daily Fitbit tables (activity_summary, heart_rate_summary, and
  related summaries) were aggregated to weekly means aligned to the same
  windows.

  Training shard produced (exemplar run):
    • K = 3 interventions (metformin, atorvastatin, lisinopril)
    • F = 25 features (EHR + wearable channels)
    • N ≈ 4.3k valid patient–intervention examples with roughly balanced arm
      counts
    • Exports: patient_tensors.npz, feature_vocab.json, intervention_map.json,
      manifest.json (reproducibility)

  Note: observational EHR and optional wearables imply missingness handled via
  masks; claims are associational without an external causal identification
  strategy.


3. JEPA model (three sub-networks) and next steps
--------------------------------------------------------------------------------

  The implementation (CausalJEPA / ClinicalJEPA alias) is an I-JEPA-style
  three-module system:

    • Context encoder
        Time-series encoder (value + mask, continuous time) → pooled embedding
        s_x. Trained with backprop.

    • Target encoder
        Same architecture; produces s_y from the post window. No gradients from
        the loss; weights follow the context encoder by EMA only (momentum
        update each step).

    • Predictor + action embedding
        embed(z) in R^{z_dim} and an MLP on [s_x || embed(z)] outputs ŝ_y;
        training minimizes ||ŝ_y − s_y||² with s_y stopped (detached).

  Next steps
    (i)   Wire a full training loop on the exported AoU shards with F, K, and
          max_seq_len set from the manifest.
    (ii)  Save a checkpoint and align inference num_interventions with
          intervention_map.
    (iii) Integrate twin verification and product APIs for deployment tests.
